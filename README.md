# Project: Null Pointer

A hybrid incremental/idle resource-management game set in deep space. Built with **React Native + TypeScript + Expo**.

## Tech Stack

| Layer | Technology |
|---|---|
| Frontend / Core Logic | React Native, TypeScript |
| State Management | Zustand |
| Native Performance (Phase 2) | Java (Android background tick) |
| AI Market API (Future) | Python |

## Architecture

```
src/
├── engine/
│   ├── types.ts          # All game type definitions
│   ├── resources.ts      # Resource definitions & initial values
│   ├── GameEngine.ts     # Deterministic tick loop (framework-agnostic)
│   └── Drone.ts          # OOP drone base class + StandardDrone
├── store/
│   └── gameStore.ts      # Zustand store (syncs engine → React)
├── components/
│   ├── ResourceDisplay.tsx
│   └── HeatBar.tsx
├── screens/
│   └── MainScreen.tsx    # Main game UI
└── __tests__/
    └── GameEngine.test.ts
```

### Core Principles

1. **Deterministic Tick System** – `GameEngine` drives a fixed-interval loop (default 1 000 ms/tick). All resource generation, heat buildup, and power consumption are calculated off this central loop.
2. **OOP Drone Logic** – `BaseDrone` (abstract) → `StandardDrone`. New drone types extend `BaseDrone` and override `mine()`.
3. **Separation of Concerns** – `GameEngine` is 100% framework-agnostic. Zustand subscribes to tick events and propagates state to React components.
